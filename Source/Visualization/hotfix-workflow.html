Hotfix Workflow

Start a new hotfix:

Development of new hotfixes start from the "master" branch.

Optionally can be started from a [base] branch. More on that later.

git flow hotfix start [version] [base]

Work on your hotfix:

Work on your hotfix, and create one or more commits.

git add .
git commit -am "That is MUCH less broken."

Finish your hotfix:

Finish your hotfix. This action:

  • Merges your hotfix into develop.
  • Merges your hotfix into master.
  • Tags the master branch with your [version].

git flow hotfix finish [version]
git push
git push --tags
